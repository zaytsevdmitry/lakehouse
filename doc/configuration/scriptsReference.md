# Ссылки на скрипты (script references)
Скрипт - динамическая часть сервиса, позволяющая адаптировать и кастомизировать функционал. Наиболее частое применение в моделях [датасетов](datasets.md),  
[sql шаблонах](sqlTemplate.md), тестовых наборах [метрик качества данных](qualityMetric.md)  в виде специального объекта - ссылки.
Применение ссылок вместо прямого использования текста скриптов позволяет применять:
* переиспользование скриптов
* составные скрипты. 
* большие скрипты без необходимости экранировать управляющие символы текста.


## Поля объекта
 Поле                          | Назначение                                                                                                                                      |
|:------------------------------|:------------------------------------------------------------------------------------------------------------------------------------------------|
| key                           | Уникальный идентификатор, имя или путь к файлу. При использовании пути к файлоу необходимо заменять "/" на другой символ. На пример на "."      | 
| order                         | Указывает исполняющему коду в каком порядке должен быть поставлен скрипт, на пример при использовании [коллекции скриптов](#Коллекция-скриптов) |

> Примечание: сортировку коллекции выполняет API сервиса конфигурирования, но способ и коннечный вариант применения остается за исполняющим движком. (processor или processorBody)

[ScriptReferenceDTO.java](../../lakehouse-common/src/main/java/org/lakehouse/client/api/dto/configs/ScriptReferenceDTO.java)


# Коллекция скриптов
Может применяться для формирования составного скрипта, где каждый скрипт будет использован в виде фрагмента. Фрагмент может быть переиспользован для многих состравных скриптов
