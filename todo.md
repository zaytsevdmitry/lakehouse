# Неочевидное переопределение datasource.service.properties + taskExecutorAttrs 
Объект datasource.service.properties является произвольным списком параметров типа ключ - значение.
в task есть объект taskExecutorAttrs, тоже ключ - значение.
В логике taskProcessor происходит объединение некоторых ключей для целей самого taskpocessor.
Эта логика может быть неочевидна пользователю. Возможно стоит привести эти объекты к одному классу и ввести общие правила объединения, тогда , при наличии четких закономерностей и понятного нейминга пользователю легче понимать как это управляется
